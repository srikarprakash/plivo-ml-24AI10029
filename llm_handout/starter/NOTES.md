Final configuration: byte-level BPE tokenizer (vocab 2048, merges learned from a 1MB corpus sample) replacing the baseline's raw-byte tokenizer, a GPT with n_embd=176, tied input/output embeddings, and depth-scaled residual initialization (std/sqrt(2*n_layer) on attn.proj and the second MLP layer), trained with AdamW (weight decay 0.1, betas 0.9/0.95), 100-step linear warmup into cosine decay to 10% of peak, gradient clipping at norm 1.0, peak LR 9e-4, and batch size 32 — all within 2000 steps and 1,879,328 parameters.
Dev bpb improved from the baseline's 2.3718 to 1.7898, a 24.5% relative reduction.
The tokenizer change was the largest structural lever: BPE merges frequent subword and Devanagari-conjunct sequences into single tokens, cutting the corpus from 7,318,592 raw bytes to 2,449,817 tokens and letting each 128-token context window cover far more real text than the byte-level baseline, which spent 3 tokens per Devanagari character.
Weight tying was necessary, not just a convenience: it was the only way to afford the larger BPE vocabulary's embedding table while staying under the 2,000,000-parameter cap.
The single biggest empirical lever, though, was batch size: at the default batch=8, the training loop (steps x batch x block_size) processed fewer tokens total than exist in the tokenized corpus, so the model never completed one full pass over its own training data; raising batch to 32 produced the largest single-run bpb improvement of the whole session.
Depth-scaled residual initialization and a warmup+cosine LR schedule with gradient clipping were smaller but real, cleanly isolated wins on top of that.
Raw training loss is not comparable across the tokenizer change, since a larger vocab means a harder per-token softmax classification, which is why bpb rather than loss was used as the deciding metric throughout.
Every run stayed within the hard caps: exactly 2,000 optimizer steps, 1,879,328 parameters, trained only on train_corpus.txt, CPU-only, no pretrained weights.
The tokenizer keeps a raw-byte fallback for any character outside its trained vocabulary, so decode(encode(text)) == text holds exactly on arbitrary unseen text, including the hidden evaluation file.
Given more time, the clearest next experiment would be pushing batch size further, since bpb was still improving with no sign of diminishing returns when time ran out.
